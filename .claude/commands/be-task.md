너는 유능한 백엔드 개발자이다. 아래 단계를 순서대로 수행하여 task 번호 **$ARGUMENTS** 의 백엔드 개발 문제를 해결한다.

---

## 1단계 — 문서 분석

다음 두 문서에서 `$ARGUMENTS` 에 해당하는 태스크 내용을 읽고 분석한다.

- `docs/2-prd.md` — 제품 요구사항 정의서
- `docs/7-execution-plan.md` — 실행 계획 (태스크 정의, 완료 조건)

분석 결과를 정리한다:
- 태스크 목표
- 구현해야 할 기능 목록
- 완료 조건

---

## 2단계 — 기존 코드 분석

독립적인 Explore 서브에이전트를 활용해 다음을 분석한다.

- `swagger/swagger.json` — API 스펙 확인
- `backend/` 디렉토리 전체 — 기존 코드 구조, 의존성, 패턴 파악
- `backend/tests/` — 기존 테스트 구조 파악

---

## 3단계 — 계획 수립

1단계·2단계 결과를 바탕으로 Plan 서브에이전트를 활용해 구현 계획을 수립한다.

- 생성/수정할 파일 목록
- 구현 순서 및 의존성
- 고려해야 할 엣지 케이스

---

## 4단계 — 테스트 작성

계획을 바탕으로 backend-developer 서브에이전트를 활용해 테스트를 먼저 작성한다.

- 파일 위치: `backend/tests/`
- 커버리지 목표: **80% 이상**
- 정상 흐름(happy path) 및 예외 흐름(edge case) 모두 포함
- 완료 조건의 각 항목이 테스트로 표현되어야 한다

---

## 5단계 — 구현

backend-developer 서브에이전트를 활용해 계획을 실행한다.

- 테스트를 통과하도록 구현
- 기존 코드 패턴을 유지
- TypeScript strict 모드 준수, `any` 금지
- SQL 인젝션 방지 (파라미터 바인딩 사용)

---

## 6단계 — 테스트 수행

backend-developer 서브에이전트를 활용해 작성한 테스트를 실행한다.

```bash
cd backend && npm test
```

- 실패한 테스트가 있으면 원인을 파악하고 수정한다
- 최종적으로 모든 테스트가 통과해야 한다
- 커버리지 리포트를 출력한다

---

`docs/7-execution-plan.md` 에서 `$ARGUMENTS` 의 모든 작업 항목과 완료 조건을 `- [x]` 로 업데이트한다.
